<Section id="loads-y-dumps" title="json.loads() y json.dumps()">

`json.loads()` convierte un string JSON a tipos Python. `json.dumps()` hace lo inverso. Los tipos que JSON soporta nativamente: `str`, `int`, `float`, `bool`, `None`, `list`, `dict`.

<CodeBlock
  tabs={[
    {
      lang: "py",
      label: "Python",
      code: `import json

# json.loads(): JSON string → Python
raw = '{"name": "Ana", "age": 30, "active": true, "score": null}'
data = json.loads(raw)

print(type(data))           # <class 'dict'>
print(data["name"])         # Ana
print(data["active"])       # True (bool Python, no "true" JSON)
print(data["score"] is None) # True

# json.dumps(): Python → JSON string
user = {
    "name": "Carlos",
    "age": 25,
    "tags": ["python", "web"],
    "address": {"city": "Madrid", "country": "España"},
}

json_str = json.dumps(user)
print(json_str)
# {"name": "Carlos", "age": 25, "tags": ["python", "web"], ...}

# Tipos Python → JSON
print(json.dumps(True))   # true
print(json.dumps(None))   # null
print(json.dumps([1, 2])) # [1, 2]

# Parsear una respuesta HTTP simulada
api_response = '''
{
    "status": "success",
    "data": [
        {"id": 1, "title": "Post uno"},
        {"id": 2, "title": "Post dos"}
    ],
    "total": 2
}
'''
response = json.loads(api_response)
for post in response["data"]:
    print(f"#{post['id']}: {post['title']}")`
    }

<Section id="load-dump-archivos" title="json.load() y json.dump() con archivos">

`json.load(file)` lee JSON desde un objeto de archivo abierto. `json.dump(data, file)` escribe JSON directamente a un archivo — más eficiente que `dumps()` + write manual.

<CodeBlock
  tabs={[
    {
      lang: "py",
      label: "Python",
      code: `import json
import tempfile
import os

config = {
    "database": {
        "host": "localhost",
        "port": 5432,
        "name": "myapp"
    },
    "debug": False,
    "allowed_hosts": ["localhost", "127.0.0.1"],
}

# Escribir al archivo con json.dump()
tmp_path = tempfile.mktemp(suffix=".json")

with open(tmp_path, "w", encoding="utf-8") as f:
    json.dump(config, f)

print(f"Archivo creado: {os.path.getsize(tmp_path)} bytes")

# Leer del archivo con json.load()
with open(tmp_path, "r", encoding="utf-8") as f:
    loaded = json.load(f)

print(loaded["database"]["host"])  # localhost
print(loaded["allowed_hosts"])     # ['localhost', '127.0.0.1']
print(loaded == config)            # True

os.unlink(tmp_path)

# Patrón común: función helper para leer/escribir config
def read_json(path: str) -> dict:
    with open(path, "r", encoding="utf-8") as f:
        return json.load(f)

def write_json(path: str, data: dict) -> None:
    with open(path, "w", encoding="utf-8") as f:
        json.dump(data, f, ensure_ascii=False, indent=2)`
    }
  ]}
  output={"Archivo creado: 89 bytes\nlocalhost\n['localhost', '127.0.0.1']\nTrue"}
/>

</Section>

<Section id="indent-sort-keys" title="indent y sort_keys para pretty print">

Los parámetros opcionales de `json.dumps()` controlan el formato del JSON generado. `indent` añade sangría para legibilidad humana; `sort_keys` ordena las claves alfabéticamente.

<CodeBlock
  tabs={[
    {
      lang: "py",
      label: "Python",
      code: `import json

data = {
    "nombre": "Python",
    "version": 3.12,
    "features": ["type hints", "pattern matching", "async"],
    "meta": {"author": "GvR", "year": 1991},
}

# Sin formato (compacto, para transmisión)
compact = json.dumps(data, separators=(",", ":"))
print(compact[:60])  # {"nombre":"Python","version":3.12,...

# Con indentación (legible, para archivos de configuración)
pretty = json.dumps(data, indent=2, ensure_ascii=False)
print(pretty)

# Con claves ordenadas (útil para diffs y comparaciones)
ordered = json.dumps(data, indent=2, sort_keys=True, ensure_ascii=False)

# ensure_ascii=False: permite caracteres no-ASCII (acentos, etc.)
spanish = {"ciudad": "Córdoba", "país": "España"}
print(json.dumps(spanish))                              # con escapes \u...
print(json.dumps(spanish, ensure_ascii=False))          # Córdoba, España

# Comparar tamaños
print(f"Compacto: {len(compact)} caracteres")
print(f"Pretty:   {len(pretty)} caracteres")`
    }
  ]}
  output={"{\n  \"nombre\": \"Python\",\n  \"version\": 3.12,\n  ...\n}\nCórdoba, España\nCompacto: ~95 caracteres\nPretty:   ~120 caracteres"}
/>

</Section>

<Section id="json-encoder-custom" title="JSONEncoder custom para tipos no serializables">

Por defecto, `json.dumps()` falla con `datetime`, `Decimal`, `set`, `dataclass` y objetos custom. Un `JSONEncoder` personalizado resuelve esto sin convertir manualmente antes de serializar.

<CodeBlock
  tabs={[
    {
      lang: "py",
      label: "Python",
      code: `import json
from datetime import datetime, date
from decimal import Decimal
from dataclasses import dataclass, asdict
from typing import Any

# Tipos no serializables por defecto
try:
    json.dumps(datetime.now())
except TypeError as e:
    print(f"Error: {e}")  # Object of type datetime is not JSON serializable

# JSONEncoder custom
class SmartEncoder(json.JSONEncoder):
    def default(self, obj: Any) -> Any:
        if isinstance(obj, datetime):
            return obj.isoformat()          # "2026-05-03T15:30:00"
        if isinstance(obj, date):
            return obj.isoformat()          # "2026-05-03"
        if isinstance(obj, Decimal):
            return float(obj)               # Decimal('29.99') → 29.99
        if isinstance(obj, set):
            return sorted(list(obj))        # set → list ordenada
        if hasattr(obj, "__dataclass_fields__"):
            return asdict(obj)              # dataclass → dict
        return super().default(obj)

@dataclass
class Product:
    name: str
    price: Decimal
    tags: set
    created_at: datetime

p = Product(
    name="Python Handbook",
    price=Decimal("29.99"),
    tags={"python", "tutorial", "dev"},
    created_at=datetime(2026, 5, 3, 10, 0),
)

result = json.dumps(p, cls=SmartEncoder, indent=2, ensure_ascii=False)
print(result)`
    }
  ]}
  output={"Error: Object of type datetime is not JSON serializable\n{\n  \"name\": \"Python Handbook\",\n  \"price\": 29.99,\n  \"tags\": [\"dev\", \"python\", \"tutorial\"],\n  \"created_at\": \"2026-05-03T10:00:00\"\n}"}
/>
