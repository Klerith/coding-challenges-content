<Callout title="¿Cuándo comentar?">
Comenta el **porqué**, no el qué. El código ya explica qué hace; el comentario debe explicar por qué es necesario o qué problema resuelve. `// incrementa contador` sobre `i++` es ruido — `// usamos i++ porque el índice es 1-based en esta API` es valioso.
</Callout>

</Section>

<Section id="prettier" title="Formato automático con Prettier">

Prettier es un formateador de código que convierte esto:

<CodeBlock
  tabs={[
    {
      lang: "js",
      label: "Antes de Prettier",
      code: `const  obj={nombre:"Ana",edad:25,activo:true}
function   saludar(   n ){return "Hola "+n}
const arr=[1,2,  3,4,5]`
    }
  ]}
/>

en esto:

<CodeBlock
  tabs={[
    {
      lang: "js",
      label: "Después de Prettier",
      code: `const obj = { nombre: "Ana", edad: 25, activo: true };
function saludar(n) {
  return "Hola " + n;
}
const arr = [1, 2, 3, 4, 5];`
    }
  ]}
/>

Configura Prettier con `.prettierrc`:

<CodeBlock
  tabs={[
    {
      lang: "json",
      label: ".prettierrc",
      code: `{
  "semi": true,
  "singleQuote": true,
  "trailingComma": "es5",
  "printWidth": 100,
  "tabWidth": 2
}`
    }
  ]}
/>

</Section>
