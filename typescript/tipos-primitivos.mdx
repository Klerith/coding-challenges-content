<Section id="primitivos" title="¿Qué es un tipo primitivo?">

Los valores primitivos son **inmutables** y se comparan por valor. Son la materia prima de todo programa JavaScript.

| Tipo | Ejemplo | Descripción |
|---|---|---|
| `string` | `"hola"` | Texto |
| `number` | `42`, `3.14` | Enteros y decimales |
| `boolean` | `true`, `false` | Verdadero o falso |
| `null` | `null` | Ausencia intencional de valor |
| `undefined` | `undefined` | Variable no asignada |
| `bigint` | `9007199254740992n` | Enteros muy grandes |
| `symbol` | `Symbol("id")` | Identificador único |

En el día a día usarás los primeros cinco. `bigint` y `symbol` aparecen en casos específicos.

</Section>

<Section id="number" title="number: números">

JavaScript usa un solo tipo para enteros y decimales (IEEE 754 de 64 bits).

<CodeBlock
  tabs={[
    {
      lang: "js",
      label: "JavaScript",
      code: `const entero = 42;
const decimal = 3.14;
const negativo = -100;
const grande = 1_000_000; // separador visual (ES2021)

// Operaciones
console.log(0.1 + 0.2); // 0.30000000000000004 ← punto flotante!

// Métodos útiles
console.log(Math.round(3.7));   // 4
console.log(Math.max(1, 5, 3)); // 5
console.log(Number.isInteger(42));    // true
console.log(Number.isFinite(1 / 0)); // false
console.log(parseInt("42px"));        // 42
console.log(parseFloat("3.14em"));    // 3.14`
    },
    {
      lang: "ts",
      label: "TypeScript",
      code: `const precio: number = 19.99;
const cantidad: number = 5;

function calcularTotal(p: number, q: number): number {
  return p * q;
}

calcularTotal(precio, cantidad); // 99.95
calcularTotal("19.99", 5);       // ❌ Error de tipo`
    }
  ]}
/>

<Callout variant="warn" title="El problema de 0.1 + 0.2">
En JavaScript `0.1 + 0.2 === 0.3` es **false**. Para dinero y cálculos de precisión, usa `Math.round(valor * 100) / 100` o una librería como `decimal.js`.
</Callout>

</Section>

<Section id="null-undefined" title="null vs undefined">

Ambos representan la ausencia de valor, pero con una semántica distinta.

  headers={["", "null", "undefined"]}

  ]}
/>

<CodeBlock
  tabs={[
    {
      lang: "js",
      label: "Diferencia práctica",
      code: `let usuario = null;    // "no hay usuario aún" (intencional)
let nombre;            // undefined (no asignado)

console.log(usuario);  // null
console.log(nombre);   // undefined

// Comparación
console.log(null == undefined);  // true  (igualdad débil)
console.log(null === undefined); // false (igualdad estricta)`
    },
    {
      lang: "ts",
      label: "TypeScript",
      code: `// Con strict: true, debes manejar null y undefined explícitamente
function saludar(nombre: string | null): string {
  if (nombre === null) return "Hola, invitado";
  return \`Hola, \${nombre}\`;
}

// Optional chaining para null/undefined
const usuario: { nombre?: string } = {};
console.log(usuario.nombre?.toUpperCase()); // undefined (no error)`
    }
  ]}
/>

<Callout title="Nullish coalescing ??">
El operador `??` devuelve el operando derecho solo si el izquierdo es `null` o `undefined` (a diferencia de `||` que también reacciona a `0`, `""`, `false`):
`const valor = resultado ?? "default";`
</Callout>

</Section>
